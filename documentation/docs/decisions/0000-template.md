## Notes for the agent writing ADRs

- One ADR per genuinely architectural decision. Not every implementation choice.
- If you find yourself writing an ADR for "should I use camelCase or snake_case," delete it. Style decisions go in a coding standards doc.
- Number ADRs sequentially: 0001, 0002, etc.
- Use kebab-case-titles in the filename: `0007-z3-timeout-behavior.md`.
- "Status: Proposed" is the default. The user moves it to "Accepted" at a checkpoint review.
- Do not edit an Accepted ADR. To change a decision, write a new ADR that supersedes it.
